## Project Overview

This project builds a full **predictive maintenance analytics pipeline** using the
NASA C-MAPSS (Commercial Modular Aero-Propulsion System Simulation) dataset —
the industry-standard benchmark for turbofan engine health monitoring, cited in
1,000+ academic papers.

**The business question:** *Can we predict when an aircraft engine will fail —
and by how much — using sensor data alone?*

## Dataset: NASA C-MAPSS

| Property | Value |
|---|---|
| Source | NASA Ames Prognostics Centre of Excellence |
| Citation | Saxena & Goebel (2008) |
| Engines | 100 (training) + 20 (test) |
| Readings | 21,654 rows (training) |
| Sensors | 21 channels (14 informative) |
| Operating modes | 6 discrete flight conditions |
| Target variable | RUL = cycles remaining to failure |

**Download options:**
- Kaggle: `kaggle datasets download -d behrad3d/nasa-cmaps`
- NASA directly: https://phm-datasets.s3.amazonaws.com/NASA/6.+Turbofan+Engine+Degradation+Simulation+Data+Set.zip
- Place `.txt` files in `data/raw/` — the scripts auto-detect and fall back to synthetic data if missing.

## Cleaning Steps (02_data_cleaning.py)

| Step | Method | Impact |
|---|---|---|
| Schema validation | Enforce dtypes, drop invalid cycles | Data integrity |
| Null imputation | Forward-fill per engine, median fallback | No data loss |
| Constant sensor removal | CV < 0.001 threshold | Drop 7 of 21 sensors |
| Outlier clipping | IQR × 3.0 per sensor | Robust statistics |
| RUL labelling | max_cycle - current_cycle, capped at 125 | Target variable |
| Rolling features | Mean + std over 5/10/20-cycle windows | +84 features |
| Health Index | Min-max normalised composite of 14 sensors | Interpretable KPI |
| Operational regimes | Rule-based binning of op_settings | 6 labels |
| Health stages | RUL-based cut: Healthy/Early/Moderate/Severe/Critical | Classification target |

## Key Features Engineered

- **RUL_capped** — Piece-wise linear RUL with 125-cycle saturation (industry standard)
- **health_index** — Normalised composite degradation score (0 = new, 1 = failed)
- **health_stage** — 5-level categorical label for Power BI filtering
- **Rolling mean/std** — 5, 10, 20-cycle temporal smoothing for 14 sensors
- **cycle_pct** — Normalised life position enabling cross-engine comparison
- **op_regime** — Operational cluster for stratified analysis
- **Anomaly flags** — Rapid degradation detection and critical RUL alerts
